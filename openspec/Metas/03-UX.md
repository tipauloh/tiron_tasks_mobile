# Metas — 03 · UX

## Princípios
Inspiração: Apple Fitness (anéis/score), Things 3 (calma), Todoist (rapidez),
Sunsama (foco diário), Linear (clareza). Visual limpo, mínimo de texto, poucos toques,
feedback imediato, foco em progresso, confortável em telas pequenas e uma mão.

## Cores de progresso (thresholds)
- `≥ 70%` → `success` (#10B981)
- `≥ 40%` → `warning` (#F59E0B)
- `< 40%` → `danger` (#EF4444)
Score/tendência usam `primary` (#208AEF) como cor de marca.

## Fluxo: Atualização rápida (a ação mais importante)
1 toque no KPI/KR → abre **QuickUpdateSheet** (BottomSheet) com o valor atual já
preenchido e teclado numérico em foco → digita → **Salvar**. Fecha e recalcula.
Meta: **≤ 3 toques, < 10 s**.

## Fluxo: Criar meta (assistido, leve)
Categoria (chips) → Nome → Prazo (atalhos hoje/semana/mês/custom via CalendarPicker) →
Resultados-Chave (1–5; cada um: título + tipo de KPI + alvo + unidade) → salvar.
Passos curtos, um foco por tela/seção.

## Componentes de UX
- **ScoreCard** — anel circular grande + % + status + TrendIndicator.
- **GoalCard** — nome, %, ProgressBar, prazo (dias restantes).
- **KpiStat** — rótulo + valor formatado pelo tipo (kg, %, R$, h…).
- **TrendIndicator** — ⬆ Melhorando / ➡ Estável / ⬇ Atenção (sem gráfico).
- **CategoryPicker / KpiTypePicker** — BottomSheet (padrão ListSelector).
- **QuickUpdateSheet** — atualização rápida.
- **ProgressBar / CircularProgress** — novos no design system, reutilizáveis.

## Critérios de aceite de UX (testáveis)
- Atualizar um KPI em **≤ 3 toques**.
- Entender o dashboard em **< 5 s** (score + status + tendência acima da dobra).
- Navegação previsível: dashboard → toque → atualizar/abrir; "+" cria.
- Sem dashboards corporativos, sem excesso de gráficos/cores/configuração.
